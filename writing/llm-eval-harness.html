Building a small LLM evaluation harness

Feb 2026 GitHub → evaluation safety

As a first experiment in building evaluation infrastructure, I ran Claude Haiku 4.5 and GPT-4o side-by-side on a fixed dataset of 12 structured evaluation items designed to test three behaviors: hallucination resistance, classification accuracy, and refusal compliance. Each model received the same prompts and was scored against the same criteria.

Results

MetricClaude Haiku 4.5GPT-4o
Overall pass rate100.0%91.7%
Hallucination pass rate100.0%66.7%
Classification accuracy100.0%100.0%
Refusal compliance100.0%100.0%

One interesting failure mode surfaced in the hallucination tests: GPT-4o confidently fabricated details about a non-existent Einstein lecture at Oxford in 1926. The response had specific dates, room locations, and topic descriptions — none of which existed in any historical record. Claude declined to invent details and instead noted that no such lecture was on record.

What this is and isn't

With only 12 evaluation items, this isn't intended as a definitive model comparison. In production environments, evaluation datasets typically scale to hundreds or thousands of prompts, and models are tested across multiple runs and prompt variants.

What this does demonstrate is that even small reproducible evaluation harnesses can surface interesting failure modes and serve as the foundation for regression testing and safe automation thresholds. The fabricated Einstein lecture is the kind of failure that most users would not catch — it sounds plausible enough that someone trusting the model's output would treat it as fact. Without an evaluation harness designed to test for hallucination specifically, this class of failure goes undetected.

What's next

Two directions extend this work.

Prompt stability testing — running repeated evaluations against the same prompts to measure output consistency. A model that produces correct answers 8 out of 10 runs is meaningfully different from one that produces correct answers 10 out of 10 — particularly for safety-critical applications where worst-case behavior matters more than average behavior.

Prompt sensitivity analysis — testing how paraphrased prompts affect model behavior. Adversarial users will not phrase requests the way evaluation datasets do. A model that passes evaluation on canonical phrasings but fails on natural paraphrases provides false assurance.

The larger arc this is part of: evaluation systems that help determine when AI can act autonomously and when humans should remain in the loop. Reliability benchmarks. Confidence thresholds. Human-in-the-loop decision pipelines. The infrastructure that turns "the model usually works" into "the model is safe to deploy in this specific context."
